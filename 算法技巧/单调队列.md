
单调队列（Monotonic Queue）是数据结构中非常精妙的设计。它在普通双端队列（Deque）的基础上，增加了一个**硬性约束**：**队列内的元素必须保持单调递增或单调递减。**

---

### 1. 核心性质

#### ① 队首的'最值'性

单调队列最显著的性质是：**队首元素永远是当前候选集合中的最值。**

- 如果是单调递减队列，队首就是**最大值**。
    
- 如果是单调递增队列，队首就是**最小值**。
    
    这使得获取最值的时间复杂度为 **$O(1)$**。
    

#### ② 元素的“生存逻辑”

单调队列维护的是一种“又强又年轻”的逻辑：

- 如果元素 $A$ 在元素 $B$ 之后进入队列（$A$ 更年轻），且 $A$ 比 $B$ 更优（比如求最大值时 $A > B$），那么 $B$ 永远失去了成为最值的机会。
    
- **性质：** 凡是比新进入元素“弱”且比它“老”的元素，都会被立即从队尾弹出（淘汰）。
    

---

### 2. 操作性质：两端开口，按需出入

虽然单调队列使用的是双端队列（Deque），但它的进出规则非常严格：

- **队尾进（Push）**：所有新元素都从队尾进入。进入前，为了保持单调性，会不断弹出队尾比它“弱”的元素。
    
- **队尾出（Pop）**：如上所述，用于维护单调性。
    
- **队首出（Pop）**：当队首元素在滑动窗口中“过期”（下标超出了范围）时，从队首弹出。
    


---

### 总结


- **单调性**：队列内元素严格/非严格单调。
    
- **及时性**：过期的元素（窗口外）必须从队首滚蛋。
    
- **高效性**：$O(1)$ 查最值，$O(n)$ 扫全集。

# 手动维护

是的，**必须由你手动维护**。

单调队列并不是 Java 或 C++ 标准库里直接提供的一个类（比如 `MonotonicQueue`），它是一种**算法思想**。

通常会使用双端队列 `Deque`（在 Java 中常用 `ArrayDeque` 或 `LinkedList`）作为底层容器，然后通过编写逻辑来手动保证它的单调性。

### 1. 需要手动维护的“两条战线”

维护单调队列就像管理一个排队的队伍，你需要同时盯着**队头**和**队尾**：

#### 第一：在队尾维护“单调性” (Add 操作)

正如你刚才看到的 `5421` 变成 `543` 的过程，这是最关键的手动操作。

- **逻辑**：在插入元素前，用一个 `while` 循环不断剔除队尾那些比新元素“弱”的旧元素。
    
- **目的**：确保队列里留下的永远是潜在的“最强者”。
    

#### 第二：在队头维护“有效性” (Poll/Pop 操作)

既然是滑动窗口，旧的元素总会滑出窗口。

- **逻辑**：当你的窗口向右移动，且窗口最左侧的下标已经超过了队首元素的下标时，你需要手动 `pollFirst()`。
    
- **目的**：确保队首的“最强者”不是已经过期的。
    

---

### 2. 手动维护的标准代码模板

以 **LeetCode 239. 滑动窗口最大值** 为例，这就是你手动维护单调性的“标准姿势”：
Java
```
Deque<Integer> deque = new ArrayDeque<>(); // 存储的是下标，方便判断过期
for (int i = 0; i < nums.length; i++) {
    // 1. 手动维护队尾单调性：把比当前元素小的全部踢走
    while (!deque.isEmpty() && nums[deque.peekLast()] < nums[i]) {
        deque.removeLast();
    }
    
    // 2. 将当前元素下标入队
    deque.addLast(i);
    
    // 3. 手动维护队头有效性：如果队首元素滑出窗口了，踢走
    if (deque.peekFirst() <= i - k) {
        deque.removeFirst();
    }
    
    // 4. 此时队首就是当前窗口的最大值
    if (i >= k - 1) {
        res[index++] = nums[deque.peekFirst()];
    }
}
```
