# 题目


给定两个字符串 `s` 和 `t`，长度分别是 `m` 和 `n`，返回 s 中的 **最短窗口 子串**，使得该子串包含 `t` 中的每一个字符（**包括重复字符**）。如果没有这样的子串，返回空字符串 `""`。

测试用例保证答案唯一。

**示例 1：**

**输入：** s = "ADOBECODEBANC", t = "ABC"
**输出：**"BANC"
**解释：** 最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。

**示例 2：**

**输入：** s = "a", t = "a"
**输出：**"a"
**解释：** 整个字符串 s 是最小覆盖子串。

**示例 3:**

**输入:** s = "a", t = "aa"
**输出:** ""
**解释:** t 中两个字符 'a' 均应包含在 s 的子串中，
因此没有符合条件的子字符串，返回空字符串。




# 题解

整体逻辑很长，值得慢慢看，理清每一步
- 利用ASCII码将字符出现次数存储在int数组中
- 滑动窗口的左窗口，在右窗口移动时继承，不要每次从零开始
- valid表示窗口内**满足数量要求**的字符**种类数**，（本质上是窗口内的字符种类数，加了数量要求）
- required表示need中字符种类数
- start、minLen，记录符合条件的子串



可以看着例子过一遍整体的逻辑，多过几次，感受need存储的方便，valid和required的精巧，start和minLen更新的思路，左窗口left收缩的条件，valid的+-


```java
public String minWindow(String s, String t) {
    if (s == null || t == null || s.length() < t.length()) return "";

    // 用数组代替 HashMap 记录字符出现次数，ASCII 码范围 128 足够
    int[] need = new int[128];
    int[] window = new int[128];
    
    // 统计目标字符串 t 中每个字符的需求量
    for (char c : t.toCharArray()) {
        need[c]++;
    }

    int left = 0, right = 0;
    int valid = 0; // 窗口中满足 need 条件的字符种类数
    int required = 0; // t 中不同字符的总数
    for (int i : need) {
        if (i > 0) required++;
    }

    // 记录最小覆盖子串的起始位置及长度
    int start = 0, minLen = Integer.MAX_VALUE;

    while (right < s.length()) {
        char c = s.charAt(right);
        right++; // 扩大窗口

        // 进行窗口内数据的一系列更新
        if (need[c] > 0) {
            window[c]++;
            if (window[c] == need[c]) {
                valid++;
            }
        }

        // 判断左侧窗口是否收缩（当窗口满足所有字符需求时）
        while (valid == required) {
            // 在这里更新最小覆盖子串
            if (right - left < minLen) {
                start = left;
                minLen = right - left;
            }

            char d = s.charAt(left);
            left++; // 缩小窗口

            // 进行窗口内数据的一系列更新
            if (need[d] > 0) {
                if (window[d] == need[d]) {
                    valid--;
                }
                window[d]--;
            }
        }
    }

    return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
}
```

## 一、整体思路

右指针不断向右扩大窗口，直到窗口“覆盖”了 `t` 的所有字符，然后左指针尝试缩小窗口，同时更新最小覆盖子串的答案，直到窗口不再覆盖 `t`，接着右指针继续右移……如此反复。

---

## 二、变量作用速查表

|变量|作用|
|---|---|
|`need[128]`|记录 `t` 中每个字符的需求数量（ASCII码共128个）|
|`window[128]`|记录当前窗口中各字符的数量|
|`left` / `right`|滑动窗口的左右边界，区间 `[left, right)`|
|`valid`|当前窗口中**已经满足需求**的字符种类数|
|`required`|`t` 中不同字符的种类数（即 `need` 中 >0 的位置数）|
|`start`|最小覆盖子串的起始位置|
|`minLen`|最小覆盖子串的长度，初值设为一个很大的数|

## 三、代码分段详解

### 1. 初始化与统计 `t` 的信息


```java
if(s==null||t==null||s.length()<t.length()) return "";
int[] need = new int[128];
int[] window = new int[128];
for(char c : t.toCharArray()){
    need[c]++;          // 记录 t 中每个字符的出现次数
}
int left=0, right=0, valid=0, required=0;
for(int i : need){
    if(i > 0) required++;  // 统计 t 中有多少种不同字符
}
int start=0, minLen=Integer.MAX_VALUE;
```

- `need` 数组就是需求清单，例如 `t = "ABC"` 则 `need['A']=1, need['B']=1, need['C']=1`，`required=3`。
    
- `valid` 表示当前窗口中已经**满足需求**的字符种类数，只有当窗口中某种字符的数量 **等于** `t` 中该字符的数量时，`valid` 才加 1。
    

---

### 2. 外层循环：右指针移动，扩大窗口


```java
while(right < s.length()){
    char c = s.charAt(right); // 取出将要加入窗口的字符
    right++;                  // 窗口右边界扩大
```
- `right` 始终指向**当前窗口的右边界+1**的位置，所以窗口内的字符是 `[left, right)` 这一半开区间。
    
- 将新字符加入窗口后，检查它是否为 `t` 中需要的字符：

```java
    if(need[c] > 0){
        window[c]++;                    // 窗口内该字符计数+1
        if(window[c] == need[c]){      // 如果窗口内该字符数量正好达到需求
            valid++;
        }
    }
```

只有窗口内某字符数量恰好等于需求量时，`valid` 才 +1，注意如果从小于变成等于则 `valid` 增加，如果超过则不再加，所以 `valid` 准确代表了“已满足需求的字符种类数”。

---

### 3. 内层循环：左指针移动，尝试缩小窗口并更新答案

~~~java

    while(valid == required){   // 当前窗口已经完全覆盖 t 中所有字符
        // 更新最小覆盖子串
        if(minLen > right - left){
            start = left;
            minLen = right - left;
        }
        // 准备移除左边界的字符
        char d = s.charAt(left);
        left++;                 // 窗口左边界右移，缩小窗口
        if(need[d] > 0){        // 移除的字符是 t 中需要的
            if(need[d] == window[d]){ // 如果原来窗口内该字符数量恰好等于需求
                valid--;        // 移除后需求不再满足，valid - 1
            }
            window[d]--;        // 窗口内计数 -1
        }
    }
}
~~~
- 当 `valid == required` 时，说明窗口已经覆盖了 `t` 的所有字符，此时记录一下当前窗口的长度，如果更短就更新 `start` 和 `minLen`。
    
- 接着将左边界字符移出窗口，同时更新 `window` 和 `valid`。如果移出的字符是 `t` 中需要的，并且移出前窗口内该字符数正好等于需求数，移出后就不满足需求了，所以 `valid` 必须减 1。
    

---

### 4. 返回结果

~~~java
return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
~~~
如果 `minLen` 没被更新过，说明不存在满足条件的子串，返回空字符串；否则返回这段最小子串。

---

## 关键理解点

1. **为什么用 `valid` 和 `required`？**  
    因为判断窗口是否已覆盖 `t`，不能简单地看窗口大小，而要确认每种字符的数量都达标。用 `valid` 记录“已达标字符种类数”，当它等于 `t` 中字符种类总数 `required` 时，就说明全部达标了。
    
2. **为什么窗口右边界是开区间？**  
    代码中 `right-left` 正好就是窗口长度，这是滑动窗口常见的写法，便于计算长度。
    
3. **复杂度**
    
    - 时间复杂度 O(N)：`left` 和 `right` 各遍历 `s` 一次。
        
    - 空间复杂度 O(1)：数组大小固定为 128。
