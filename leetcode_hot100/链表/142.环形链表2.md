在[[141.环形链表]]的基础上找到入环节点
注意，141中快慢指针相遇的地方不一定是入环点


# 题解

~~~java
public class Solution {
    public ListNode detectCycle(ListNode head) {
        if(head == null|| head.next == null) return null;  
        ListNode fast = head;
        ListNode slow = head;
        while(true){
            fast = fast.next.next;
            slow = slow.next;
            if(fast == null || fast.next == null) return null;
            if(fast == slow) {
                while(slow != head){
                    slow =slow.next;
                    head =head.next;
                }
                return slow;
            }
        }
    }
}

~~~

## 具体数学原理
![[Pasted image 20260324174900.png]]
如果存在环，fast和slow相遇时，fast走了2x步，slow走了x步，
设环的长度为z，则fast和slow相遇时fast比slow多走了kz步（也就是多走了k圈）
2x-x = kz  ->  x = kz
设从head到入环节点的步数为y
则slow从进环到相遇走的步数为 x-y = kz-y
kz-y+y = kz对吧，就意味着在slow相遇的时候再走y步就到入口点了（变成z的整数倍了）
而head走到入环点也是y步，
所以再相遇之后slow和head同时出发，走y步就相遇再入环点了；


### 一刷：第一次相遇前，慢指针绝对不可能走完一整圈

>提出这个问题主要是我怀疑`slow从进环到相遇走的步数为 x-y = kz-y`这句话的正确性，slow会不会走了很多圈才与fast相遇呢，这样这句话就不对了


**在环形结构中，如果慢指针（Slow）从起点出发，快指针（Fast）从环内随机位置出发，且速度比为 $1:2$，慢指针在第一次相遇前，绝对不可能走完一整圈。**

为了让你直观理解，我们用数学推导来证明这个结论。

---

### 1. 变量定义

假设：

- **$L$**：整个环的长度（节点总数）。
    
- **$d$**：初始状态下，**快指针落后于慢指针的距离**（也就是慢指针顺时针追到快指针需要的步数）。
    
    - 注意：在环中，快指针“领先”慢指针，同时也意味着它在后方“追赶”慢指针。
        
    - 由于快指针是随机分布的，这个 $d$ 的取值范围是 $[0, L-1]$。
        

### 2. 追赶过程

- 慢指针速度为 **1**，快指针速度为 **2**。
    
- 它们的**相对速度**（快指针相对于慢指针的追赶速度）是 $2 - 1 = 1$。
    
- 这意味着每过一个单位时间，快指针就会缩短与慢指针之间 **1** 个单位的距离。
    

### 3. 计算相遇时间

为了实现第一次相遇，快指针需要抹平它们之间初始的距离 $d$。

- 因为相对速度是 1，所以相遇所需要的时间 $t = d / 1 = d$。
    

### 4. 慢指针走过的路程

在时间 $t$ 内，慢指针走过的距离为：

$$S_{slow} = V_{slow} \times t = 1 \times d = d$$

### 5. 结论证明

既然我们已经知道 $d$（初始距离）的最大可能值是 $L - 1$（即快指针刚好在慢指针的前一个节点），那么：

$$S_{slow} = d \le L - 1 < L$$

**结论：** 慢指针走过的距离 $d$ 永远小于环的总长度 $L$。因此，慢指针在相遇时**连一圈都走不完**，最多只能走到离起点还差一个节点的位置。
