# 题干
给你单链表的头节点 `head` ，请你反转链表，并返回反转后的链表。

**示例 1：**

![](https://assets.leetcode.com/uploads/2021/02/19/rev1ex1.jpg)

**输入：**head = [1,2,3,4,5]
**输出：**[5,4,3,2,1]

**示例 2：**

![](https://assets.leetcode.com/uploads/2021/02/19/rev1ex2.jpg)

**输入：**head = [1,2]
**输出：**[2,1]

**示例 3：**

**输入：**head = []
**输出：**[]




# 题解
主要用到了递归的思想

## 递归

其实，递归不是靠“脑补”每一层调用是怎么运行的（那样很快就会 CPU 过载），而是靠一套**固定的公式**。

---

## 1. 递归的“三部曲”公式

每次写递归，你只需要问自己三个问题：
#### 什么时候停？

如果没有这个，程序会死循环直到栈溢出。通常是处理“最简单”的情况（比如链表为空、只有一个节点、树的叶子节点）。

- **链表反转中：** `if (head == null || head.next == null) return head;`
    

#### 递归公式（Recursive Step）—— 这一层要做什么？⭐

假设你已经拿到了“后面所有层”处理完后的结果，你现在这一层该怎么配合它？

- **注意：** 不要去想后面是怎么算的，**假定它已经算好了**。

#### ③ 返回值（Return Value）—— 往上传什么？

每一层处理完后，要把什么结果交给它的“上司”？

---

## 2. 核心心法：不要试图“跳进去”

很多同学学不会递归，是因为试图在脑子里模拟 **A调B，B调C，C调D** 的完整过程。**千万别这么做！**

**正确的做法是：**

1. **把自己当成“这一层”的负责人。**
    
2. 你的下属（递归调用）会把剩下的活儿干完并交给你一个结果。
    
3. 你只需要把剩下的那一点点活（比如反转一个指针）干完，再把总结果交给你的上司。

---

## 3. 以“反转链表”为例演练思维

我们要把 `1 -> 2 -> 3 -> 4 -> 5` 反转。

- **第一步（偷懒）：** 我是节点 `1`。我把 `2 -> 3 -> 4 -> 5` 扔给递归函数：“喂，你去帮我把后面这串反转了。”
    
- **第二步（等待）：** 递归函数（下属）非常给力，过了一会儿它返还给我一个已经反转好的链表：`5 -> 4 -> 3 -> 2`。
    
    - _注意：此时 `2` 的后面依然连着 `1` 吗？不，此时 `1` 依然指向 `2`，但 `2` 现在是反转链表的末尾了。_
        
- **第三步（干活）：** 我（节点 `1`）现在要做两件事：
    
    1. 让 `2` 回头指向我（`head.next.next = head`）。
        
    2. 我不再指向 `2`（`head.next = null`）。
        
- **第四步（交工）：** 我把整个反转好的新链表头（节点 `5`）传给我的上司。
	为什么不用想下一层该做什么呢，因为递归函数中每一层要做的事情是一样的，做好1就做好了234，至于尾部节点，我们在确定递归停止条件的时候就已经处理过了，所以处理1就好


## 代码
~~~java

class Solution {

    public ListNode reverseList(ListNode head) {

        if(head == null||head.next == null   )

            return head;

        ListNode revhead = reverseList(head.next);

        head.next.next =head;

        head.next = null;

        return revhead;

    }

}
~~~
