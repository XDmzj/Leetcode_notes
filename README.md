刷题的方法

- **分类刷题，不要乱跳**： 不要今天刷动态规划，明天刷 SQL。建议连续一周只刷“区间类”题目，你会发现所有这类题目的开头第一行几乎都是 `Arrays.sort(intervals, ...)`。当你连续写了 5 遍排序，下次看到区间，你闭着眼也会先写个排序。
    
- **不仅看解法，要看“为什么”**： 看题解时，重点看那句：**“这道题的核心矛盾在于...”**。比如合并区间，核心矛盾在于顺序是乱的，所以要排序。
    
- **刻意练习“手绘”**： 遇到数组、链表、区间题目，先别写代码。在纸上画图，像刚才那样的数轴。**图画出来了，代码通常就写出一半了。**
    
- **建立自己的“代码库”**： 把今天这个 `List<int[]>` 的用法、Lambda 排序的写法记下来。下次遇到，你就不用在语法上耗费精力，可以专心想逻辑。
